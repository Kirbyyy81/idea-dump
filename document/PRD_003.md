## Step 1 — Database Migration

**Create file:** `document/migrations/20260414_tickets.sql`

```sql
CREATE TABLE tickets (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  project_id UUID NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
  user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  title TEXT NOT NULL,
  description TEXT,
  notes TEXT,
  status TEXT NOT NULL DEFAULT 'todo'
    CHECK (status IN ('todo', 'in_progress', 'to_review', 'done', 'closed')),
  priority TEXT NOT NULL DEFAULT 'medium'
    CHECK (priority IN ('low', 'medium', 'high')),
  source TEXT NOT NULL DEFAULT 'self'
    CHECK (source IN ('self', 'user_tester')),
  tags TEXT[] DEFAULT '{}',
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

ALTER TABLE tickets ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can view own tickets" ON tickets
  FOR SELECT USING (auth.uid() = user_id);

CREATE POLICY "Users can insert own tickets" ON tickets
  FOR INSERT WITH CHECK (auth.uid() = user_id);

CREATE POLICY "Users can update own tickets" ON tickets
  FOR UPDATE USING (auth.uid() = user_id);

CREATE POLICY "Users can delete own tickets" ON tickets
  FOR DELETE USING (auth.uid() = user_id);

CREATE OR REPLACE FUNCTION update_tickets_updated_at()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = NOW();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER tickets_updated_at
  BEFORE UPDATE ON tickets
  FOR EACH ROW EXECUTE FUNCTION update_tickets_updated_at();
```

**Action required:** Run this in the Supabase SQL editor. Verify the table appears before proceeding.

## Step 3 — API Routes

### 3a. Create `app/api/tickets/route.ts`

Auth pattern: copy from `app/api/notes/route.ts` — use `createClient()` from `@/lib/supabase/server` and `supabase.auth.getUser()`. Return mock/empty data for unauthenticated users (same fallback pattern as notes). Do NOT use `resolveIdentity` — tickets are human-only, no agent API key flow needed.

```ts
// GET /api/tickets
// Query params supported:
//   project_id  (string, UUID)
//   status      (TicketStatus)
//   priority    ('low' | 'medium' | 'high')
//   source      ('self' | 'user_tester')
// Behaviour:
//   - Auth: createClient() + supabase.auth.getUser(). If no user, return { data: [] }.
//   - Query tickets table where user_id = user.id
//   - Apply each filter param if present using .eq()
//   - Order by created_at DESC
//   - Return NextResponse.json({ data })

// POST /api/tickets
// Body: CreateTicketInput
// Behaviour:
//   - Auth same as above. If no user, return mock created ticket.
//   - Validate title and project_id are present, return 400 if missing.
//   - Insert into tickets with user_id = user.id
//   - Return NextResponse.json({ data }, { status: 201 })
```

### 3b. Create `app/api/tickets/[id]/route.ts`

Mirror `app/api/logs/[id]/route.ts` structure but without the `resolveIdentity` / agent permission logic. Use the simpler `createClient()` + `getUser()` pattern.

```ts
// PATCH /api/tickets/[id]

//   - Auth: createClient() + getUser(). Return 401 if no user.
//   - Update tickets where id = params.id AND user_id = user.id
//   - Set updated_at = new Date().toISOString() (belt-and-suspenders alongside the DB trigger)

//   - Auth same as above.
//   - Delete where id = params.id AND user_id = user.id
```
