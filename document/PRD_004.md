# Article Migration Helper

## Overview

This PRD defines a new frontend-only utility page inside the personal-use tools website to speed up migration of old university articles into a new AEM website.

The page consolidates three repetitive text-transformation tasks into one workflow:

1. Minute Reader
2. Slug and Meta Image Name Generator
3. Table of Contents Anchor Generator

The goal is to reduce manual formatting time, reduce avoidable mistakes, and optimize repeated copy-paste work during article migration.

---

## Problem Statement

Current article migration work involves repeated manual text transformations:

- calculating reading time from article body content
- converting titles into lowercase dash-separated slugs
- generating AEM-friendly meta image file names
- converting multiple section headings into numbered anchor IDs

These tasks are simple but repetitive, easy to mistype, and time-consuming when performed manually across many articles.

---

## Goals

- Provide one dedicated page containing all three migration helpers.
- Make each tool work independently with instant output updates.
- Optimize for fast paste -> transform -> copy workflow.
- Keep the page frontend-only for v1 with no backend or API dependency.

---

## Shared Utility Rules

Use shared deterministic local utility functions across the page.

Suggested function signatures:

```ts
function toSlug(input: string): string
function getWordCount(text: string): number
function getReadingTime(text: string): {
  wordCount: number
  minutes: number
  label: string
}
function buildImageName(title: string, suffix: string): string
function buildTocAnchors(multilineText: string): string[]
```

### Shared Slug Rules

Both the slug generator and TOC anchor generator must use the same slug logic.

Apply in this order:

1. convert to lowercase
2. trim whitespace
3. replace `&` with `and`
4. remove apostrophes
5. replace non-alphanumeric separators with `-`
6. collapse repeated hyphens
7. remove leading/trailing hyphens

Example:

```txt
Malaysia's Health & Wellness
```

Output:

```txt
malaysias-health-and-wellness
```

Confirmed rule:

- apostrophes are removed, not converted into separators
- example: `isn't` -> `isnt`

## Tool 1: Minute Reader

### Purpose

Calculate article reading time based on pasted article content.

### Input

- multiline text area
- accepts plain text
- user pastes full article body content

### Output

- total word count
- estimated reading time label

### Calculation Rules

Word count:

- trim leading/trailing whitespace
- split on one or more whitespace characters
- ignore empty tokens
- whitespace-based count is sufficient for v1

Reading time:

```txt
readingTimeMinutes = totalWordCount / 200
```

Display logic:

- if word count is `0`, show empty output or `0 words`
- if result is greater than `0` but less than `1`, show `1 min read`
- otherwise round up to nearest whole minute

Final label format:

```txt
X min read
```

### Example

Input:

```txt
450 words of article text
```

Output:

```txt
Word count: 450
Reading time: 3 min read
```

### Acceptance Criteria

- Pasting text updates word count instantly.
- Reading time updates instantly.
- Copy button copies only the reading-time label.
- Clear button empties input and output.

## Tool 2: Slug and Meta Image Name Generator

### Purpose

Convert article titles into AEM-friendly slugs and generate full meta image filenames.

### Inputs

#### Field A: Main Title

- single-line text input
- required for meaningful output

#### Field B: Fixed Prefix

- not editable
- always:

```txt
taylors-article
```

#### Field C: Suffix Selector

- selectable suffix format
- user can change postfix depending on image type
- implemented as a dropdown or select input
- values should come from a local constant array for easy maintenance

### Outputs

#### Output 1: Basic Slug

Convert title into lowercase dash-separated slug using the shared slug function.

#### Output 2: Full Image Name

Format:

```txt
taylors-article-{slug}-{selectedSuffix}
```

### Default Suffix

Initial default:

```txt
meta-image-1200x630
```

### Initial Preset Options

```ts
export const IMAGE_SUFFIX_OPTIONS = [
  "meta-image-1200x630",
  "thumbnail-image",
  "banner-image",
  "hero-image",
]
```

These values should remain easy to edit without changing transformation logic.

### Example

Input:

- title: `Why Just Cook at Home Isn't Enough for Malaysia's Health`
- selected suffix: `meta-image-1200x630`

Output:

```txt
slug:
why-just-cook-at-home-isnt-enough-for-malaysias-health

full image name:
taylors-article-why-just-cook-at-home-isnt-enough-for-malaysias-health-meta-image-1200x630
```

### Acceptance Criteria

- Typing in title updates outputs instantly.
- Changing suffix updates the full image name instantly.
- Copy button for slug copies only the slug.
- Copy button for full image name copies only the full image name.
- Clear button clears title and resets suffix to the default.
- Fixed prefix remains `taylors-article`.

## Tool 3: Table of Contents Anchor Generator

### Purpose

Convert multiple headings into sequential anchor IDs in one pass.

### Input

- multiline text area
- each non-empty line is treated as one heading
- empty lines are ignored

### Output

- multiline text output
- each output line corresponds to one processed non-empty input line

### Required Format

For each heading:

```txt
{index}-{slugified-heading}
```

Confirmed numbering rule:

- first processed line -> `1-slug`
- second processed line -> `2-slug`
- third processed line -> `3-slug`

### Conversion Rules

For each non-empty line:

1. trim whitespace
2. convert to lowercase
3. replace `&` with `and`
4. remove apostrophes
5. replace non-alphanumeric separators with `-`
6. collapse repeated hyphens
7. trim leading/trailing hyphens
8. prepend sequential number and `-`

### Example

Input:

```txt
Introduction to Nutrition
Why Students Skip Breakfast
Healthy Eating Tips
```

Output:

```txt
1-introduction-to-nutrition
2-why-students-skip-breakfast
3-healthy-eating-tips
```

### Acceptance Criteria

- User can paste multiple headings at once.
- Output is generated instantly.
- Empty lines are ignored.
- Numbering applies only to processed non-empty lines.
- Output order matches input order.
- Copy button copies the full output block.
- Clear button clears input and output.
